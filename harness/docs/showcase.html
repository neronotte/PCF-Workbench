0Quick Start

Two environment variables control which mode the harness runs in. No config files needed.

Browse all controls in a workspace with card layout, search, and status badges.

Terminal # Gallery mode — browse all controls
cd PCFBuilderFramework/harness
PCF_WORKSPACE_ROOT="path/to/controls" npx vite --port 8181

Load a single control directly for focused development and testing.

Terminal # Single control mode
cd PCFBuilderFramework/harness
PCF_CONTROL_PATH="path/to/control" npx vite --port 8181

3Network Conditioning

Simulate real-world network conditions to test offline-first patterns and degraded connectivity.

Online0ms
OfflineThrows error
Slow 3G2000ms
Fast 3G500ms
CustomUser-defined
How it works

Every WebAPI call passes through applyNetworkConditions() before executing. In Offline mode, calls throw immediately with "Network unavailable". In throttled modes, an artificial delay is injected via setTimeout. A status badge in the toolbar shows the current network state. The custom mode lets you set any latency value in milliseconds.

4Test Scenarios

Save and restore complete harness state for reproducible testing workflows.

State Capture

Each scenario captures the full state: property values, network mode, device preset, entity context, and custom latency settings.

Manage scenarios interactively. Saved scenarios persist in localStorage and merge with file-based test-scenarios.json.

Share scenarios between team members by exporting to JSON files. Import pre-configured scenarios for CI testing or demo setups.

8Hot Reload

Instant feedback loop: edit your control, rebuild, and see changes without refreshing the browser.

File Watching

Vite watches for changes to bundle.js in the control's out/ directory. No manual browser refresh needed.

500ms Debounce

Rapid file changes are debounced to avoid redundant reloads during incremental builds.

Full Control Reinit

On reload, the control goes through the complete lifecycle: destroy() then init() + updateView(). Metrics and logs are reset.

9Thumbnail System

Automatic and manual thumbnail capture for gallery card previews.

Auto-Capture

3 seconds after the control loads, the harness captures a thumbnail using SVG foreignObject rendering.

Manual Capture

Click the camera button to capture on demand. Cross-origin content is sanitized to avoid tainted canvas errors.

Format Support

Gallery cards display .gif, .jpg, and .png thumbnails. Animated GIFs play inline for showing control interactions.

10RESX Localization

Automatic .resx file discovery and parsing for the getString() API.

How It Works

The harness parses .resx XML files referenced in the control manifest into a key-value map. When a control calls context.resources.getString(key), the localized string is returned.

  • XML parsing via fast-xml-parser
  • Extracts <data name="key"><value> entries
  • Graceful fallback on parse errors
  • Supports multiple .resx files per control
strings.resx <data name="Button_Save">
  <value>Save Changes</value>
</data>
<data name="Label_Status">
  <value>Current Status</value>
</data>

12Device API Mocks

Stubbed device APIs that return plausible test data so controls can run outside a mobile device.

captureImage

Returns a mock 1x1 transparent PNG as base64. Includes fileName, fileSize, and mimeType metadata.

getBarcodeValue

Resolves with "MOCK-BARCODE-12345". Controls using barcode scanning can test their parsing logic.

getCurrentPosition

Returns London coordinates (51.5074, -0.1278) with 10m accuracy. Controls using geolocation work seamlessly.

pickFile

Returns an empty array. Controls using file pickers can handle the "no file selected" flow.
