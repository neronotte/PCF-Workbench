StarRating

A small, accessible 0–5 star rating control for any Whole.None field. Built end-to-end via the GitHub Copilot CLI pcf-engineer + pcf-workbench skills — from plan to passing headless gate — with zero Dataverse round-trips.

v1.0.0 React + Fluent UI v9 Virtual control ~16 KiB bundle 0 leaks · 0 console errors

1Brief

StarRating binds to a Whole.None (integer) field and lets a user pick a rating between 0 and a configurable maxStars (default 5). Each star is a Fluent v9 button with full WAI-ARIA radiogroup semantics: keyboard navigation, hover preview without commit, optional “click selected star to clear” via allowClear. A dedicated isAuthoringMode preview renders a static 3-filled placeholder for the form designer.

It's deliberately small — the goal of this sample is to demonstrate the workflow, not the depth of the control. See DESIGN.md for the full spec and BUILT_WITH_AI.md for the exact prompts that produced it.

Headless render captured by pcf-harness loop. Default state, no rating set. See report.json for the full lifecycle / perf / leak report.

3How to use

1. Add to a model-driven form. Edit a numeric column (Whole Number, no format) and select “Add component” → PcfWorkbench.StarRating.

2. Configure the inputs. Two properties on top of the bound value:

value      — bound to your Whole.None column
maxStars   — total stars rendered (default 5; valid range 1–20)
allowClear — if true, clicking the selected star (or pressing 0) clears to 0

3. Test locally first. Drop the control into PCF Workbench before importing to Dataverse:

cd harness
npx tsx bin/pcf-harness.ts loop --path ..\samples\StarRating\StarRating

Expected: [summary] PASS — control rendered cleanly, exit 0, screenshot + report under ./pcf-loop-reports/.
